# YOLO para Principiantes en Vision Engine

Esta guía está pensada para alguien que viene de JavaScript, recién empieza en visión por computadora y quiere entender rápido cómo usar YOLO dentro de este proyecto.

## 1) Qué es YOLO (explicado simple)

YOLO significa **You Only Look Once**.

En términos prácticos:

1. Le pasas una imagen (o un frame de video).
2. El modelo devuelve objetos detectados.
3. Cada objeto trae:
   - clase (ejemplo: `person`)
   - confianza (`confidence` entre `0.0` y `1.0`)
   - caja (`bounding box`) con coordenadas

## 2) Conceptos mínimos

- **Inferencia**: correr el modelo sobre una imagen.
- **Clase**: categoría detectada (`person`, `car`, etc.).
- **Confidence**: qué tan seguro está el modelo.
- **Bounding Box**: rectángulo donde está el objeto.
- **Threshold**: mínimo de confidence aceptado.

Regla rápida: si subes el threshold, reduces falsos positivos pero también puedes perder detecciones reales.

## 3) Cómo se usa YOLO en este repo

Archivos clave:

- `detectors/yolo_person.py`: encapsula YOLO y filtra solo `person`.
- `app/yolo_person_preview.py`: abre webcam, corre inferencia y dibuja overlays.
- `config.py`: define defaults (modelo, resolución, confidence).
- `models/YoloClass.py`: enum con nombres de clases para evitar magic numbers.

Comando principal:

```bash
uv run vision-yolo --camera-index 0 --width 1280 --height 720 --model yolo11n-pose.pt --confidence 0.45 --tracker-config bytetrack.yaml
```

Resultado esperado en pantalla:

- Bounding boxes para personas.
- Confidence por detección.
- FPS + cantidad de personas detectadas.

## 3.1 Dónde está el filtro de clase

En este proyecto el filtro de clase está en `detectors/yolo_person.py`, donde se compara contra `YoloClass.PERSON`.

Eso evita usar el valor `0` directo y hace el código más legible.

## 4) Modelos YOLO más conocidos

En este proyecto usamos YOLO vía Ultralytics (`ultralytics`), típicamente familia `yolov8`.

Variantes comunes:

- `yolo11n-pose.pt` (nano pose): más rápido, incluye keypoints.
- `yolov8s.pt` (small): balance liviano.
- `yolov8m.pt` (medium): balance general recomendado cuando tienes más recursos.
- `yolov8l.pt` / `yolov8x.pt`: más precisos, más pesados.

Elección práctica:

1. Empieza con `yolo11n-pose.pt` para validar flujo con keypoints.
2. Si necesitas más precisión y tu máquina aguanta, prueba `yolov8s.pt` o `yolov8m.pt`.

## 5) Instalación y setup

### Dependencias

```bash
uv venv
uv pip install --python .venv/bin/python -e ".[dev]"
```

Esto instala OpenCV, Ultralytics y herramientas de calidad.

### Descarga de pesos

Ultralytics descarga automáticamente el peso la primera vez que usas `--model yolo11n-pose.pt`.

Si quieres fijar otro modelo por variable de entorno:

```bash
VISION_YOLO_MODEL=yolov8s.pt uv run vision-yolo
```

Si quieres cambiar threshold por entorno:

```bash
VISION_YOLO_CONFIDENCE=0.35 uv run vision-yolo
```

## 6) Dónde buscar modelos

Fuentes recomendadas:

1. Ultralytics oficial (primera opción).
2. Hugging Face (modelos comunitarios, revisar calidad).
3. Roboflow Universe (datasets/modelos de casos específicos).

Recomendación: para MVP y estabilidad, prioriza pesos oficiales de Ultralytics.

## 7) Troubleshooting rápido

### No detecta nada

1. Baja threshold (ejemplo `0.30`).
2. Mejora iluminación.
3. Asegúrate de estar usando una clase soportada por el modelo.

### Va lento

1. Usa `yolo11n-pose.pt`.
2. Baja resolución (`--width 960 --height 540`).
3. Evita ejecutar procesos pesados en paralelo.

### Error de dependencia

Reinstala editable con dev:

```bash
uv pip install --python .venv/bin/python -e ".[dev]"
```

## 8) Tu siguiente paso recomendado

1. Ejecuta `vision-yolo` y ajusta confidence hasta que te guste el balance.
2. Guarda una configuración base en variables de entorno.
3. Avanza a modelado de detecciones (Hito A.5) separando aún más dominio vs proveedor YOLO.
